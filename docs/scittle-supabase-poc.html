Stock Scittle from jsDelivr, no fork, no build step. ClojureScript namespaces are fetched async from a simulated database, topologically sorted by their dependencies, evaluated in order via scittle.core.eval_string, and rendered through Reagent. Click simulate edit to mutate a namespace at runtime and see the change reflected without a page refresh.

    Honest note on architecture choice.
    The earlier design doc proposed :async-load-fn via
    sci.async. That path turns out to require a custom Scittle build
    — stock Scittle bundles only sci.core and does not expose
    register-plugin! to JavaScript. The strategy used here
    (JS-side topological sort + sequential eval_string) achieves
    the same architectural goal — code source-of-truth in the database, loaded
    on demand at runtime — and works against the unmodified jsDelivr bundle.
    The trade-off is that (:require ...) inside DB-loaded code
    only resolves namespaces already evaluated, so the loader must topo-sort
    upfront rather than lazily resolve per-require.
